# FAQ

## 一般

### ブックマークデータはどこに保存されますか？
ローカルの SQLite データベース（`userData/bookmarks.db`）に保存されます。詳細はセットアップガイドの「データの保存場所」を参照してください。

### データベースを手動でバックアップしたい
データベースファイル（`bookmarks.db`）を直接コピーしてください。アプリが起動していない状態で行うことを推奨します。

## 同期について

### 同期データは GitHub に公開されますか？
リポジトリを **非公開** で作成すれば、あなた以外はアクセスできません。公開リポジトリでも構いませんが、ブックマークのURLが公開されることに留意してください。

### PAT の権限は最小限で済みますか？
`repo` スコープが必要です（コンテンツの読み書きのため）。トークンはアプリのローカル設定にのみ保存され、GitHub には送信されません（API 経由の認証のみに使用）。

### 複数デバイスで同時に同期したらどうなりますか？
各デバイスが独立したファイルに保存されるため、同時同期でも問題ありません。同じブックマークが別々に更新された場合のみ、次回同期時にコンフリクトが検出されます。

### デバイスを削除したい
該当デバイスの `device-{id}.json` をリポジトリから削除し、`metadata/devices.json` から該当エントリを削除してください。

## インポートについて

### インポート後にブラウザのブックマークは変更されますか？
いいえ。本ツールはブラウザのデータを「読み取り」しません。ブラウザからエクスポートした HTML ファイルを取り込む形のため、ブラウザ側のデータに影響はありません。

### フォルダ構造は保持されますか？
HTML ファイル内のフォルダ構造はタグとしてインポートされます。ツール内ではフラットな一覧として管理されます。

## トラブルシューティング

### アプリが起動しない
```bash
pnpm rebuild    # better-sqlite3 を再ビルド
pnpm dev        # 再試行
```

### 同期が「接続失敗」になる
- GitHub のトークンが有効か確認（トークンの有効期限が切れていないか）
- リポジトリ名・所有者が正しいか確認
- ネットワーク接続を確認

### 重複が検出されない
- URL の正規化後も区別が必要なページの場合、手動で判断してください
- 同一ドメインでもクエリパラメータが重要な場合は、正規化ルールの調整を検討してください

### AI アシスタントが応答しない
- `http://localhost:9876/api/health` にアクセスしてサーバが起動しているか確認
- Zed の AI 機能が有効か確認
- ファイアウォールでローカルポートがブロックされていないか確認
